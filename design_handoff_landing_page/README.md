# Handoff: لندینگ پیج «رنگ زنگ» (روح رنگ بابل)

## Overview
یک لندینگ‌پیج تک‌صفحه‌ای فارسی/RTL برای معرفی محصول «رنگ زنگ» (پوشش ضدخوردگی صنعتی که مستقیم روی سطوح زنگ‌زده اجرا می‌شود)، ساخته‌شده برای شرکت صنایع روح رنگ بابل. صفحه شامل هدر با هالهٔ نوری روی بک‌گراند عکس صنعتی، معرفی محصول، ویژگی‌ها، راهنمای اجرا، گواهی‌نامه‌ها، گالری نمونه‌کار با لایت‌باکس، بنر CTA، فرم تماس و فوتر است.

**هدف نهایی:** ساخت یک پروژهٔ React + Vite واقعی و ریسپانسیو (دسکتاپ/تبلت/موبایل) از روی این طرح.

## About the Design Files
فایل‌های این پکیج **مرجع طراحی هستند که با HTML ساخته شده‌اند** — پروتوتایپ‌هایی برای نشان‌دادن ظاهر و رفتار نهایی، نه کدی برای کپی مستقیم در پروژه. وظیفهٔ توسعه‌دهنده این است که این طرح‌های HTML را **در محیط React + Vite بازسازی کند**: کامپوننت‌بندی درست (Nav، Hero، TrustStrip، Solution، Features، Guide، Certifications، Gallery+Lightbox، CTABanner، ContactForm، Footer)، تبدیل state به React hooks (`useState`)، و پیاده‌سازی ریسپانسیو با CSS واقعی (media query / clamp) به‌جای سه نسخهٔ جدا برای هر بریک‌پوینت.

فایل مرجع اصلی، `reference-design-all-variants.dc.html`، شامل **چند نسخهٔ اکسپلوریشن** از این پروژه است (گزینه‌های 1a/1b قدیمی‌تر هم داخلش هست). **فقط به بخش‌های زیر توجه کنید** — این سه، سه بریک‌پوینت (دسکتاپ/تبلت/موبایل) از **همان یک طرح نهایی** هستند، نه گزینه‌های متفاوت:

- `id="1c"` → نسخهٔ **دسکتاپ** (عرض کانواس مرجع: 1440px) — سکشن‌ها با پسوند `-c` (مثلاً `#hero-c`)
- `id="2a"` → نسخهٔ **تبلت** (عرض کانواس مرجع: 834px، قاب تبلت) — سکشن‌ها با پسوند `-t` (مثلاً `#hero-t`)
- `id="2b"` → نسخهٔ **موبایل** (قاب آیفون، عرض داخلی ~390px) — سکشن‌ها با پسوند `-m` (مثلاً `#hero-m`)

بخش‌های `1a`، `1b` و هر چیز دیگری در فایل را **نادیده بگیرید** — قدیمی و بی‌ربط به نسخهٔ نهایی هستند.

`reference-ios-frame.jsx` صرفاً قاب تزئینی آیفون برای نمایش پروتوتایپ موبایل در مرورگر است — **بخشی از سایت نهایی نیست** و در پروژهٔ Vite لازم نمی‌شود.

## Fidelity
**High-fidelity.** رنگ‌ها، تایپوگرافی، فاصله‌گذاری و ساختار در همهٔ سه بریک‌پوینت نهایی و پیکسل‌دقیق هستند. کپی متن‌ها (فارسی) هم نهایی است، مگر جایی که صراحتاً به‌عنوان placeholder علامت‌گذاری شده (تصاویر محصول/گالری/لوگوهای گواهی‌نامه).

## Global Setup
- **جهت و زبان:** کل صفحه `dir="rtl" lang="fa"`.
- **فونت:** `Vazirmatn` (فونت فارسی، وزن‌های regular/medium/bold استفاده شده — باید به‌صورت وب‌فونت لود شود، مثلاً از Google Fonts یا self-hosted).
- **پس‌زمینهٔ پایه صفحه:** `oklch(98% 0.005 90)` (سفید مایل به کرم گرم).
- تمام رنگ‌ها با `oklch()` تعریف شده‌اند؛ اگر تارگت پروژه از این فرمت پشتیبانی نمی‌کند، با ابزار تبدیل رنگ (مثلاً culorijs) به hex/rgb تبدیل کنید بدون تغییر ارزش رنگ.

## Screens / Views

### 1) Desktop (نسخهٔ 1c — عرض مرجع 1440px)

**Nav** — سیستیکی، ارتفاع 88px، بک‌گراند `oklch(18% 0.015 240)` (نیوی تیره)، پدینگ افقی 64px. سمت راست (چون RTL، منطقاً سمت چپ بصری): لوگو متنی «رنگ زنگ» (23px/700/`oklch(97% 0.01 90)`) + زیرنویس «از صنایع روح رنگ بابل» (12px/`oklch(75% 0.02 220)`). سمت مقابل: 6 لینک ناوبری (14px/`oklch(85% 0.015 220)`) + دکمهٔ CTA پیل «مشاوره رایگان» (بوردر 1.5px رنگ طلایی `oklch(74% 0.11 85)`، متن هم‌رنگ، بدون فیل).

**Hero** (`#hero-c`) — پدینگ `150px 40px 0`، بک‌گراند: عکس صنعتی (سازهٔ فلزی/کارخانه) با گرادیان تیرهٔ روی آن (`linear-gradient(180deg, oklch(15% .015 240 / .82), oklch(18% .015 240 / .94))` روی `url(...) center/cover`). دو هالهٔ نوری بلرشده (radial-gradient، فیلتر blur 50px) در گوشهٔ پایین-چپ: یکی آبی‌فیروزه‌ای (`oklch(58% 0.09 195 / .55)`, عرض900×ارتفاع420) و یکی طلایی (`oklch(74% 0.11 85 / .5)`, عرض600×ارتفاع360)، هر دو overflow:hidden داخل هیرو. محتوا: max-width 720px، مرکزچین — لیبل کوچک (14px، لتر-اسپیسینگ 2px، رنگ فیروزه‌ای روشن)، H1 «رنگ زنگ» (82px/700/رنگ تقریبا سفید)، ساب‌هدلاین طلایی (23px/500)، پاراگراف توضیح (16.5px، لاین‌هایت 2.1، رنگ خاکستری‌روشن، max-width 580px)، دو دکمهٔ پیل کنار هم (outline طلایی + fill نیمه‌شفاف فیروزه‌ای).

**Trust strip** — نوار باریک زیر هیرو، بک‌گراند روشن، 3 آیتم متنی جداشده با «·» (بیش از ۱۰ سال سابقه / تاییدیه ICS کانادا / ISO 9001:2008).

**Solution** (`#solution-c`) — پدینگ 110px 64px، max-width 1160px. H2 مرکزچین 38px + پاراگراف توضیح مرکزچین. گرید 2 ستونه (سنتی در برابر رنگ‌زنگ): کارت راست بوردردار خالی (روش سنتی، لیست 4 آیتمه)، کارت چپ بک‌گراند طلایی کم‌رنگ (`oklch(93% 0.04 85 / .5)`, بدون بوردر، همون لیست استایل). زیرش یک باکس تصویر محصول تمام‌عرض (ارتفاع 300px، الان placeholder striped pattern با caption مونواسپیس — باید عکس واقعی قوطی محصول جایگزین شود).

**Features** (`#features-c`) — H2 مرکزچین، گرید 3×2 (6 آیتم)، هر آیتم: دایرهٔ شماره (48px، بوردر یک‌درمیان طلایی/فیروزه‌ای، بدون فیل)، تیتر بولد 17px، توضیح 14px رنگ خاکستری.

**Guide** (`#guide-c`) — بک‌گراند کرم گرم `oklch(93% 0.02 90)`. گرید 4 ستونه از کارت‌های مرحله (بک‌گراند سفید تقریبا خالص، رادیوس 24px، پدینگ 32px) + یک کارت مشخصات فنی زیرشان (گرید 3 ستونه از 6 آیتم متریک).

**Certifications** (`#cert-c`) — H2 + پاراگراف مرکزچین، گرید 6 ستونه از بج‌های دایره‌ای (56px، بوردر یک‌درمیان طلایی/فیروزه‌ای، حرف مخفف داخلش) + کپشن زیر هرکدام. **این بج‌ها الان متن‌placeholder هستند و باید با لوگوهای واقعی گواهی‌نامه جایگزین شوند.**

**Gallery** (`#gallery-c`) — H2 مرکزچین، گرید 4 ستونه از تامبنیل (ارتفاع 210px، رادیوس 24px، الان striped-pattern placeholder با کپشن — باید عکس واقعی نمونه‌کار جایگزین شود). کلیک روی هرکدام → **لایت‌باکس مودال**: overlay تیرهٔ تمام‌صفحه، کارت مرکزی سفید (max-width 900px)، دکمهٔ بستن (✕) گوشهٔ بالا-چپ، تصویر بزرگ (ارتفاع 480px)، ناوبری قبلی/بعدی + شمارندهٔ `index/total` زیر تصویر. کلیک روی overlay = بستن؛ کلیک روی خود کارت = بدون اثر (stopPropagation).

**CTA banner** — نوار تیرهٔ تمام‌عرض (پدینگ 80px 64px)، یک هالهٔ نوری فیروزه‌ای بلرشده در بالا-وسط، تیتر 28px + توضیح + دکمهٔ پیل تماس تلفنی (`tel:`).

**Contact form** (`#contact-c`) — گرید 2 ستونه نامتقارن (1.1fr / 0.9fr). ستون راست: کارت بوردردار با فرم (نام+تلفن کنار هم، نوع پروژه، توضیحات textarea، دکمهٔ ارسال پیل outline). ستون چپ: کارت بک‌گراند طلایی کم‌رنگ با اطلاعات تماس مستقیم (آدرس، تلفن، موبایل، ایمیل، وبسایت).

**Footer** — بک‌گراند نیوی تیره، گرید 4 ستونه (درباره / دسترسی سریع / تماس / شبکه‌های اجتماعی) با بوردر جداکننده پایین، کپی‌رایت مرکزچین ته صفحه.

### 2) Tablet (نسخهٔ 2a — عرض مرجع 834px)
همان ساختار دقیقاً به‌صورت **فشرده‌شده و تک‌ستونه‌ترشده**: Nav ارتفاع 72px با پدینگ 28px و فونت‌های کوچک‌تر (لینک‌ها 12.5px). Hero پدینگ `90px 28px 0`، H1 56px، پاراگراف max-width 460px، دکمه‌ها هنوز کنار هم اما فونت 14px. Solution/Features/Guide همان گرید 2 ستونه‌ها می‌مانند اما Features از 3×2 به **2×3** تغییر می‌کند و Guide از 4 ستونه به **2×2**. Certifications از 6 ستونه به **3×2**. Gallery از 4 ستونه به **2×2**. Contact form از گرید 2 ستونهٔ نامتقارن به **تک‌ستونه (فرم روی هم اطلاعات تماس)**. Footer از 4 ستونه به **2×2**. لایت‌باکس max-width 700px، تصویر ارتفاع 340px.

### 3) Mobile (نسخهٔ 2b — قاب آیفون، عرض داخلی ~390px)
Nav داخل هدر ثابت (پدینگ‌بالا 58px برای فضای status bar شبیه‌سازی‌شده)، فقط لوگو + **دکمهٔ همبرگر (☰)** به‌جای لینک‌های افقی. با کلیک روی همبرگر، یک **منوی کشویی عمودی** (بک‌گراند نیوی تیره‌تر، پوزیشن sticky زیر هدر) باز می‌شود با 6 لینک + دکمهٔ CTA پر («مشاوره رایگان»، بک‌گراند طلایی پر، متن تیره)؛ کلیک روی هر لینک منو را می‌بندد. Hero پدینگ `36px 20px 0`، H1 38px، دکمه‌های CTA **زیر هم به‌جای کنار هم** (تمام‌عرض). همهٔ گریدهای بخش‌های دیگر **تک‌ستونه** می‌شوند (Solution، Features، Guide) به‌جز Certifications و Gallery که **2 ستونه** می‌مانند. Contact form تک‌ستونه، فرم روی هم اطلاعات تماس. Footer تک‌ستونه با گروه‌های stack‌شده. لایت‌باکس max-width 340px، تصویر ارتفاع 220px، دکمه‌های قبلی/بعدی کوچک‌تر.

## Interactions & Behavior
- **ناوبری اسکرول:** همهٔ لینک‌های Nav/Footer به `id`های سکشن‌ها با anchor (`#hero-c`, `#features-t`, و...) اسکرول می‌کنند — در پیاده‌سازی واقعی این باید smooth-scroll باشد.
- **منوی موبایل:** state بولین `mobileNavOpen`؛ کلیک روی دکمهٔ همبرگر toggle می‌کند؛ کلیک روی هر لینک داخل منو آن را می‌بندد (`false`).
- **Gallery + Lightbox:** هر آیتم گالری با کلیک، لایت‌باکس مربوط به همان بریک‌پوینت را باز می‌کند (state شامل `{ variant, index }` — واریانت مشخص می‌کند کدام لایت‌باکس/بریک‌پوینت باز است). دکمه‌های «قبلی»/«بعدی» ایندکس را با wrap-around (modulo) عوض می‌کنند. دکمهٔ ✕ یا کلیک روی overlay آن را می‌بندد؛ کلیک روی خود کارت مودال باعث بسته‌شدن نمی‌شود.
- **Hover:** آیتم‌های گالری روی هاور کمی محو می‌شوند (`opacity: 0.82`, transition 0.15s). دکمه‌های دیگر hover state اختصاصی ندارند در پروتوتایپ — پیشنهاد می‌شود در پیاده‌سازی واقعی حالت hover ساده (تیره‌ترشدن بوردر/بک‌گراند) به همهٔ دکمه‌های پیل اضافه شود.
- **فرم تماس:** غیرفعال در پروتوتایپ (بدون منطق ارسال واقعی) — فیلدها: نام و نام خانوادگی، شماره تماس، نوع پروژه (فقط دسکتاپ)، توضیحات پروژه (textarea)، دکمهٔ ارسال. باید submit handler واقعی (API call / mailto) اضافه شود.
- **تماس تلفنی:** دکمه‌ها و متن‌های تلفن با `tel:` لینک شده‌اند (`tel:09023353317`, `tel:02125632356`).
- **ریسپانسیو:** سه نسخه باید در پیاده‌سازی نهایی با یک کدبیس واحد و CSS ریسپانسیو (media query) پیاده‌سازی شوند، نه سه صفحهٔ جدا — نسخه‌های 1c/2a/2b صرفاً مرجع بریک‌پوینت دسکتاپ/تبلت/موبایل هستند.

## State Management
- `mobileNavOpen: boolean` — کنترل باز/بستهٔ منوی موبایل.
- `lightbox: { variant, index } | null` — کدام لایت‌باکس (بر اساس بریک‌پوینت) باز است و کدام آیتم گالری (۰ تا ۳) نمایش داده می‌شود؛ `null` یعنی بسته.
- داده‌های گالری از یک آرایهٔ ثابت 4 آیتمی می‌آید: `[{caption: 'پالایشگاه نفت', tone:'teal'}, {caption:'اسکله و سازه دریایی', tone:'gold'}, {caption:'سازه فولادی صنعتی', tone:'teal'}, {caption:'ناوگان خودرویی سنگین', tone:'gold'}]` — در پیاده‌سازی واقعی این باید با URL عکس واقعی جایگزین شود، نه رنگ/کپشن.
- بدون data fetching در پروتوتایپ — همه‌چیز استاتیک/کلاینت-ساید.
